# llmlingua-2

Typed LLMLingua-2 prompt compression for Node, via `@huggingface/transformers`.

## Build Commands

| Task | Command |
|---|---|
| Test (unit) | `just test` |
| Test (real model, ~560 MB download) | `just integration` |
| Lint | `just lint` |
| Format | `just fmt` |
| Docs | `just docs` |
| Build | `just build` |
| Pre-release smoke | `just release-preflight` |

## Architecture Overview

Single-package layout. Public API is exposed via `src/index.ts` and bundled by
tsup to `dist/` as dual ESM (`index.js`) + CJS (`index.cjs`) with `.d.ts`
declarations for both.

The wrapper is a thin, typed envelope around the LLMLingua-2 ONNX export hosted
on the Hugging Face Hub. The model is loaded lazily on the first `compress`
call and reused for subsequent calls in the same process.

## Key Modules

| Module | Purpose | Notes |
|---|---|---|
| `src/index.ts` | Package entry / public re-exports | All JSDoc here flows into the docs API reference |
| `src/wrapper.ts` | `LLMLingua2Wrapper` class + default singleton | Public API surface |
| `src/pipeline.ts` | `loadModel` — tokenizer + model construction via transformers.js | Handles `revision`, `quantized`, `transformersOptions` |
| `src/chunking.ts` | `splitForXlmR` — sentence-bounded chunker for the 512-token window | English-biased; documented v0.1 limitation |
| `src/errors.ts` | `LLMLingua2Error` hierarchy with stable `.code` strings | `ENOT_AVAILABLE`, `EINVALID_REVERSE_MAP` |
| `src/types.ts` | Public type aliases (options, results, reverseMap) | |
| `src/version.ts` | Generated by `scripts/gen-version.mjs` from `package.json` | Do not edit by hand |

## Conventions

- **Testing:** vitest. Unit tests in `tests/*.unit.test.ts` mock transformers.
  Real-model integration lives in `tests/integration.test.ts` and self-gates via
  `describe.skipIf(!process.env.LLMLINGUA_INTEGRATION)`.
- **Docstrings:** JSDoc. TypeDoc generates the docs API reference from these
  comments via the `starlight-typedoc` plugin.
- **Formatting authority:** `biome format`. `biome check --write` for safe
  lint fixes. Biome config disables `noNonNullAssertion` and
  `noAssignInExpressions` (intentional patterns in src/), and `noExplicitAny`
  for tests (mock typing).
- **Pre-commit guard:** `biome check --write` on every commit; `tsc --noEmit`
  on push. See `.pre-commit-config.yaml`.

## The justfile is the canonical command surface

`npm` and `tsup` are implementation details. Add new dev workflows as `just`
recipes, not as ad-hoc shell commands in README.

## Public API contract — DO NOT BREAK

`LLMLingua2Wrapper` implements the `LLMLinguaWrapper` interface pinned by
upstream consumers (pi-mesh-g). The contract is enforced by
`tests/contract.test.ts`. Treat the public surface as frozen:

- `modelId`, `version`, `available` readonly properties
- `constructor(options?: WrapperOptions)`
- `compress(text, opts?) → Promise<{ compressed, reverseMap }>`
- `decompress(compressed, reverseMap) → Promise<string>`

The error hierarchy (`LLMLingua2Error`, `LLMLingua2NotAvailableError`,
`LLMLingua2InvalidReverseMapError`) and their `.code` strings are also part of
the contract.

## Releases

Bump `version` in `package.json` (and update `CHANGELOG.md`) in a dedicated PR.
When that PR merges to `main`, `tag-on-version-bump.yml` detects the bump and
pushes a `vX.Y.Z` tag, which triggers `release.yml` to publish to npm with
provenance.

## Integration suite

`tests/integration.test.ts` downloads ~560 MB of ONNX weights from the Hugging
Face Hub on first run and caches them. NOT a PR gate. Runs weekly (Mondays
07:00 UTC) and on `workflow_dispatch` via `.github/workflows/integration.yml`.

Local: `LLMLINGUA_INTEGRATION=1 just integration`.
